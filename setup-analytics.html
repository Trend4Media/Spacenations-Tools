📋 Setup-Schritte

1. Firebase-Sammlungen erstellen

Erstellt die notwendigen Firestore-Sammlungen für Analytics-Daten:

  • analytics_pageViews - Seitenaufrufe
  • analytics_sessions - Benutzer-Sessions
  • analytics_events - Benutzer-Events

2. Sicherheitsregeln einrichten

Konfiguriert Firestore-Sicherheitsregeln für Analytics-Daten:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Analytics-Sammlungen - nur für authentifizierte Benutzer match /analytics_pageViews/{document} { allow read, write: if request.auth != null; } match /analytics_sessions/{document} { allow read, write: if request.auth != null; } match /analytics_events/{document} { allow read, write: if request.auth != null; } } }

🔧 Manuelle Konfiguration

Falls das automatische Setup fehlschlägt, können Sie die Konfiguration manuell durchführen:

Firebase Console

1. Öffnen Sie die Firebase Console

2. Wählen Sie Ihr Projekt aus

3. Gehen Sie zu "Firestore Database"

4. Erstellen Sie die folgenden Sammlungen:

  • analytics_pageViews
  • analytics_sessions
  • analytics_events

Sicherheitsregeln

1. Gehen Sie zu "Firestore Database" → "Regeln"

2. Fügen Sie die oben gezeigten Regeln hinzu

3. Klicken Sie auf "Veröffentlichen"
